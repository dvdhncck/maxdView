Currently known limitations and problems

Please report any problems, comments or suggestions direct to the developer, david.hancock@cs.man.ac.uk

  • Clusters:

    clusters are not re-mapped to rows when a file is loaded [FIXED, mostly, be careful with existing clusters when merging data from different array types]

  • Code Runner:

    there is no method of killing the code fragment if it doesn't terminate itself. [FIXED]

    The JIT compiler interface doesn't work (yet)

    the parsing of error messages to make the line numbeers correspond to the code fragment doesn't work properly and make the -------^ indicators point to the wrong thing

  • Cluster Manager:

    the tree is completely rebuilt rather than updated ater file operations. This is annoying because all branches collapse. [FIXED]

  • Drag and Drop: On Linux there are problems with the Custom Menu interfering with Drag and Drop. If you use the left mouse button to selecte something from the Custom menu, no subsequent Drag and Drop operations will work. If you stick exclusively to the right mouse button for the custom menu then things are ok. This is not a problem on MS-Windows.

  • Sort Clusters:

    the brute force algorithm (which kicks in if the special case algorithm has failed) can take a _very_ long time to complete (but it will complete) [FIXED, algorithm replaced with much better one]

  • Rescan Plugins:

    sometimes there is a very long pause after the rescan, resizing the window makes everything work again straight away. Not sure about this, something to do with the way the menus are built?. [FIXED]

  • Space Plot popup menus get lost in the image canvas. It can also use vast amounts of memory.
